# Attempts at bootstrapping GHC, packaged in Nix

## Current state

For each major release of GHC since 6.2, its latest patch version X.Y.Z can be built by running `nix build .#ghc_X_Y_Z` on an x86-64 Linux machine (except for 9.12.3 being packaged instead of 9.12.4 and 9.14.1 not being packaged).
Running `nix build` without arguments defaults to 9.12.3, the latest packaged version.

For each packaged version, the official source distribution is built against the official binary distribution of the same version.
Indeed, the first challenge for an old GHC version is to compile it against itself but with a modern toolchain, before trying to compile it against another Haskell compiler, which should be harder.

## Known limitations

Builds use pre-existing GHC binaries.
The first goal is to compile a modern GHC version without using any pre-existing Haskell compiler/interpreter binary anywhere in the whole bootstrap chain.

Futhermore, the official source distributions come with pre-generated files.
Pre-generated files are not allowed in a clean bootstrap.
Among them are lexers and parsers pre-generated by Alex and Happy, both Haskell binaries.
The second goal is to get rid of pre-generated files in the bootstrap chain, or at least the ones pre-generated by Haskell binaries.
